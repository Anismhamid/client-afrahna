Project Overview

The `server-afrahna` project is a robust backend API built with Node.js and Express. It serves as the core data and logic provider for the Afrahna application, managing various aspects of events, users, and bookings.

It leverages MongoDB as its database through Mongoose, ensuring flexible and scalable data storage. Key features include JWT-based authentication, user management, event creation and management, and booking functionalities.

Key Technologies

  • Node.js & Express.js: Fast, unopinionated, minimalist web framework.
  • MongoDB & Mongoose: Flexible NoSQL database and elegant MongoDB object modeling for Node.js.
  • JWT (JSON Web Tokens): For secure and stateless authentication.
  • Bcryptjs: For hashing passwords securely.
  • Dotenv: For managing environment variables.

Getting Started

Follow these steps to set up and run the `server-afrahna` project locally.

Prerequisites

  • Node.js (LTS version recommended)
  • npm (comes with Node.js) or Yarn
  • MongoDB instance (local or cloud-hosted)

Installation

  1. Clone the repository:

    git clone https://github.com/Anismhamid/server-afrahna.git

  2. Navigate into the project directory:

    cd server-afrahna

  3. Install dependencies:

    npm install

    or

    yarn install

Environment Variables

Create a `.env` file in the root of the project with the following variables: 

PORT=5000
MONGO_URI=mongodb://localhost:27017/afrahna-db
JWT_SECRET=YOUR_VERY_STRONG_JWT_SECRET
JWT_EXPIRES_IN=1h
  • PORT: The port the server will run on.
  • MONGO_URI: Your MongoDB connection string.
  • JWT_SECRET: A secret key for signing JWTs. **Use a strong, unique value for production.**
  • JWT_EXPIRES_IN: The expiration time for JWTs (e.g., `1h`, `7d`).

Running the Server

Once all dependencies are installed and the `.env` file is configured, you can start the server: 

npm start

The server will typically start on the port specified in your `.env` file (e.g., `http://localhost:5000`).

API Endpoints

The `server-afrahna` API provides the following endpoints. All successful responses will typically include a `status` and `message` field, along with the requested `data`.

Authentication API (/api/auth)

Handles user registration and login, returning JWT tokens for authenticated requests.

POST /api/auth/register

Registers a new user.

Request Body: 
{
  "username": "string",
  "email": "string",
  "password": "string"
}
Response (Success 201): 
{
  "status": "success",
  "message": "User registered successfully",
  "data": {
    "_id": "string",
    "username": "string",
    "email": "string"
  }
}

POST /api/auth/login

Authenticates a user and returns a JWT token.

Request Body: 
{
  "email": "string",
  "password": "string"
}
Response (Success 200): 
{
  "status": "success",
  "message": "Logged in successfully",
  "data": {
    "token": "string",
    "user": {
      "_id": "string",
      "username": "string",
      "email": "string"
    }
  }
}

Users API (/api/users)

Manage user profiles. Requires authentication for most operations.

GET /api/users/:id

Retrieve a user by ID. Authentication Required

Response (Success 200):

  "data": {

    "username": "string",
    "email": "string",
    "createdAt": "Date",
    "updatedAt": "Date"

}

Update a user's profile. Authentication Required

Request Body (Partial):

{
  "status": "success",
  "message": "User updated successfully",

    "_id": "string",
    "username": "string",

DELETE /api/users/:id

Delete a user. Authentication Required

Response (Success 200): 
{
  "status": "success",
  "message": "User deleted successfully"
}

Database Models

The `server-afrahna` project uses Mongoose schemas to define the structure of its data in MongoDB. Here's a high-level overview of the primary models:

User Model

Represents a user of the application.

  • username: String, unique, required.
  • email: String, unique, required.
  • password: String, required (hashed).
  • createdAt: Date, automatically set.
  • updatedAt: Date, automatically set.

Event Model

Defines an event created by an organizer.

  • title: String, required.
  • description: String.
  • date: Date, required.
  • location: String.
  • price: Number, required.
  • capacity: Number, required.
  • organizer: ObjectId (ref to User), required.
  • createdAt: Date, automatically set.
  • updatedAt: Date, automatically set.

Booking Model

Records a user's booking for an event.

  • user: ObjectId (ref to User), required.
  • event: ObjectId (ref to Event), required.
  • tickets: Number, required.
  • bookingDate: Date, required (when the booking was made).
  • createdAt: Date, automatically set.
  • updatedAt: Date, automatically set.
